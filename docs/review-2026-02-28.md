## Critical（3 项）

### C-1. `config-guard.js` 的 `exit 2` 对 ConfigChange 事件无效

- **文件**：`config-guard.js` 第 39 行
- **问题**：根据 Hook I/O 协议，`exit 2 + stderr` 仅对 Stop 和 PreToolUse 生效。ConfigChange 事件不支持 `exit 2` 阻止操作。当前代码意图是阻止 `disableAllHooks=true`，但实际上 Claude Code 只会将其视为 hook 执行错误，配置变更仍会生效。核心防护功能形同虚设。
- **修复建议**：ConfigChange 无法阻止操作，只能通过 `exit 0 + JSON stdout additionalContext` 发出强烈警告。或者改用 PreToolUse 拦截写入 settings 文件的 Edit/Write 操作。

### C-2. `stop.js` walkthrough 日期正则只匹配第一个日期

- **文件**：`stop.js` 第 86 行
- **问题**：`String.match()` 不带 `g` 标志时返回第一个匹配。如果活跃区有多个工作记录条目（从上到下时间递增），第一个匹配到的是最旧的日期。导致几乎永远报"今天的工作尚未记录"的持续误报。
- **修复建议**：改用 `matchAll` 取最后一个日期：
  ```js
  const allDates = [...walkActive.matchAll(/\*\*(?:追加)?时间\*\*:\s*(\d{4}-\d{2}-\d{2})/g)];
  const lastDate = allDates.length > 0 ? allDates[allDates.length - 1][1] : null;
  ```

### C-3. Bash 工具完全绕过 PACE 保护

- **文件**：`settings-hooks-excerpt.json`、`pre-tool-use.js`
- **问题**：PreToolUse matcher 为 `Write|Edit`，不包含 `Bash`。AI 通过 Bash 执行 `echo > file.js`、`sed -i`、`cat >` 等命令完全绕过 PACE 检查。
- **当前缓解**：CLAUDE.md G-4 明确禁止 Bash 绕过（~70-85% 约束）
- **修复建议**：添加 PreToolUse:Bash hook，检测 stdin 中 command 字段是否包含文件写入模式，发出 additionalContext 警告或 deny。

## High（1 项）

### H-1. `change-implementation_plan.md` 格式与 hooks 正则不兼容

- **文件**：`skills/templates/change-implementation_plan.md`
- **问题**：使用 Emoji+表格格式（📝/🔄/✅），而 hooks 所有正则基于 Checkbox 列表格式（`- [ ]`/`- [/]`/`- [x]`）。如果通过此模板创建 implementation_plan.md，hooks 的 E 阶段前提检查、CHG 完成检测、Stop 一致性检查全部失效。
- **修复建议**：废弃或重写为与 `artifact-implementation_plan.md` 一致的 Checkbox 格式。

## Warning（14 项）

### W-1. `config-guard.js` disableAllHooks 正则过于宽松
- **文件**：`config-guard.js` 第 36 行
- **问题**：`/disableAllHooks.*true/i` 匹配注释、文档等假阳性
- **修复**：改为 `/"disableAllHooks"\s*:\s*true/`

### W-2. `config-guard.js` PACE hook 删除检测假阳性高
- **文件**：`config-guard.js` 第 43 行
- **问题**：`/\/pace\//i.test(configStr) && /delete|remove/i.test(configStr)` 任何含 delete 词的配置都触发
- **修复**：检查结构化 JSON 而非全文字符串匹配

### W-3. `pace-utils.js` frontmatter 正则不兼容 CRLF
- **文件**：`pace-utils.js` 第 155 行
- **问题**：`/^---\n/` 不匹配 `\r\n`，Obsidian notes 若为 CRLF 则 scanRelatedNotes 静默失效
- **修复**：改为 `/^---\r?\n/`

### W-4. `pace-utils.js` mklink /J 命令注入风险
- **文件**：`pace-utils.js` 第 105 行
- **问题**：`cmd /c mklink /J` 路径直接拼入 shell，特殊字符可能注入
- **修复**：改用 `fs.symlinkSync(cwd, target, 'junction')` 或 `execFileSync`

### W-5. `stop.js` walkthrough 无日期时不产生警告
- **文件**：`stop.js` 第 84-93 行
- **问题**：`lastDate` 为 null 时条件跳过，有活跃任务但 walkthrough 从未记录日期的情况被静默跳过
- **修复**：补充 `if (!lastDate && (pendingCount > 0 || xCount > 0))` 分支

### W-6. `pre-tool-use.js` isPaceProject+countCodeFiles 重复调用
- **文件**：`pre-tool-use.js` 第 106-107 行
- **问题**：`isPaceProject()` 内部已调用 `countCodeFiles()`，外部又独立调用一次，重复目录扫描
- **修复**：缓存结果或让 isPaceProject 返回 codeCount

### W-7. `pace-utils.js` VAULT_PATH 硬编码用户路径
- **文件**：`pace-utils.js` 第 9 行
- **问题**：硬编码 `C:/Users/Xiao/...`，其他用户无法使用
- **修复**：通过环境变量 `process.env.PACE_VAULT_PATH` 配置

### W-8. 日志文件无轮转/大小限制
- **文件**：所有脚本的 `pace-hooks.log`
- **问题**：持续追加无上限，长期使用可能数百 MB
- **修复**：添加大小检查，超 5MB 时截断

### W-9. `PROTECTED_ARTIFACTS` 硬编码
- **文件**：`pre-tool-use.js` 第 66 行
- **问题**：`['task.md', 'implementation_plan.md', 'walkthrough.md', 'findings.md']` 未从 ARTIFACT_FILES 派生
- **修复**：改为 `ARTIFACT_FILES.filter(f => f !== 'spec.md')`

### W-10. Skill 文件间引用路径格式错误
- **文件**：`pace-workflow.md`、`artifact-management.md`、`change-management.md`
- **问题**：引用 `skills/change-management/SKILL.md` 等不存在的路径
- **修复**：统一改为同级相对路径

### W-11. `change-management.md` Emoji 状态描述可能误导 AI
- **文件**：`change-management.md` PACE 集成表
- **问题**：用 📝/🔄/✅ 描述状态，但 hooks 匹配 Checkbox 格式
- **修复**：替换为 Checkbox 描述或明确注明 Emoji 仅为可读性

### W-12. C 阶段 APPROVED 自签风险
- **文件**：`pre-tool-use.js` H6、`post-tool-use.js` H6
- **问题**：AI 可自行添加 APPROVED 标记，PostToolUse 仅 additionalContext 警告
- **修复**：升级自签检测措辞，注入"如果未获用户确认请立即撤回标记"

### W-13. CODE_EXTS 不覆盖 C/C++/Ruby/PHP 等语言
- **文件**：`pace-utils.js` 第 7 行
- **问题**：仅 10 种扩展名，其他语言项目 code-count 信号失效
- **修复**：扩展覆盖主流语言

### W-14. E2E 测试缺失多个高风险场景
- **文件**：`tests/test-hooks-e2e.js`
- **问题**：VERIFIED 检查、E 阶段 deny、交叉验证（W9）等场景未覆盖
- **修复**：补充测试用例
