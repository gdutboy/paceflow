#### [AUDIT-03] teammate 模式下 runtime-control / marker 写入降级不一致（门序倒置）（W，合并维度2 BG-4 + 维度1 BG-2）

- **位置**: `plugin/hooks/pre-tool-use.js:705-715`（runtime-control 用 `denyOrHint`）vs `:312-321/342-350/903-913`（同级保护用 `hardDeny`）；`:1283-1322`（marker 检查 `denyOrHint` 先于 managed-artifact `hardDeny`）
- **描述**: 两处 teammate 降级缺口。(1) bash/powershell/monitor 的 artifact-runtime-control 拦截走 `denyOrHint`，teammate 模式（`CLAUDE_CODE_TEAM_NAME` 已设）降级为 additionalContext 放行；而 local-artifact-root-choice / project-root-marker / Write-Edit 写 runtime-control 都用 `hardDeny`（对 teammate 也强拦）。runtime-control 删除对任何身份都非法（deny 文案明示"只能由 hook 创建/释放"），降级与文案自相矛盾。即 team 模式下 `rm -rf .pace/change-owners`（H-2 刚修复的拦截）被绕过。(2) marker（APPROVED/VERIFIED）突变检查（`:1283`）对非 agent 用 `denyOrHint`、teammate 降级放行，而其后的 managed-artifact `hardDeny`（`:1312`）对 teammate 强拦——结果 teammate 做普通 Edit 被硬拦，但若该 Edit 恰好新增 marker 反而被降级放行，**更敏感的操作反而更宽松**。

#### [AUDIT-04] agent-lifecycle-guard 对无 action / 未知 action 的 update-chg 放行并回退 owner 状态（W，维度1 BG-3）

- **位置**: `plugin/hooks/pre-tool-use/agent-lifecycle-guard.js:316-425`；副作用点 `pre-tool-use.js:646-663`
- **描述**: `agentLifecyclePromptDenyReason` 仅对 `action∈{approve,approve-and-start,update-status,verify}` 校验必填字段；当 action 缺失或为未知值（如 `action:foo`）时所有 `mentionsX` 标志为 false，函数 `:425` `return ''`（不拦）。pre-tool-use 随后只要求 target 存在（`:566`），便写 change-owner `state='active'`（`:658`）并 PASS。
- **证据**: `promptDeclaredAction`（`:277-283`）对 `action: foo` 返回 `'foo'`；guard 中各 `mentions*` 均不命中 → 落到 `:425 return ''`。`pre-tool-use.js:646-663` 对无 action 的 update-chg 计算 `ownerState='active'` 并 `writeChangeOwner`。
- **影响**: (1) 允许派出语义不明的 update-chg（依赖 agent 自觉）；(2) 具体副作用——无 action 的 update-chg 会把既有 owner 记录从 ready/closing **回退为 active**，造成 owner 状态倒退。
- **修复建议**: guard 中对 `operation=update-chg` 增加 action 白名单校验（必须 ∈ 已知集合，否则 deny 并回模板），避免无 action 派遣与 owner 状态被动回退。

#### [AUDIT-05] bash/powershell `commandTextLooksMutating` 跨语句贪婪正则误拦只读命令（W，合并维度2 BG-2/BG-3）

- **位置**: `plugin/hooks/pre-tool-use/powershell-guard.js:113`（node/python 行）；`plugin/hooks/pre-tool-use/bash-guard.js:91`（npx 行）
- **描述**: H-1 把 bash-guard 的 node/python 行收紧为 `[^\n;&|]*`（`bash-guard.js:93`），但遗漏两处：(1) `powershell-guard.js:113` 等价 node/python 正则仍是 `[\s\S]*` 跨整串贪婪；(2) `bash-guard.js:91` 的 npx 行仍是 `[\s\S]*`，且 `-w\b` 极易命中（`grep -w`、`wc -w`、`sort` 等）。两者与 `ReferencesArtifact` 组合后误拦只读命令。
