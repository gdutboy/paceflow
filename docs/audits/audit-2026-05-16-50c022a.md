### P1 建议（短期内修）

#### [P1-1] PowerShell guard 缺 .NET 静态 API 覆盖（验证编号 V12）

- **位置**: `plugin/hooks/pre-tool-use/powershell-guard.js:75-79, 81-85`
- **问题**: `commandTextLooksMutating` 只列 cmdlet/alias 与 node/python 内联调用；`commandTextContainsWriteApi` 只查 `writeFile / appendFile / rmSync / renameSync / mkdirSync / write_text / write_bytes / open(...,'w')`。`[IO.File]::WriteAllText('task.md', 'x')`、`[System.IO.StreamWriter]::new('task.md')`、`[IO.Directory]::Delete('changes')`、`[IO.Path]::*` 等 PowerShell 调 .NET 写文件 / 删文件路径全部不触发三个 `||` 分支，PASS。
- **路径追踪**: pre-tool-use.js:717-720 合成判定 `mutatesArtifact = Redirects || EmbedsArtifactWriteScript || (LooksMutating && References)`；.NET 调用既不产生 `>` 重定向、也不在 mutating regex、也不在 WriteApi regex 中 → 三分支全 false → PASS。
- **影响**: PaceFlow 是 Windows 设计的 plugin，AI 在 Windows 主机上可能使用 PowerShell 工具；虽然 AI 不会主动选 .NET API 写文件，但语义覆盖完整性是 audit 视角下应闭合的 hole。
- **建议修复**: 在 `commandTextLooksMutating` 与 `commandTextContainsWriteApi` 中追加：
  - `\[(?:System\.)?IO\.File\]::(?:WriteAllText|WriteAllBytes|WriteAllLines|AppendAllText|AppendAllLines|Delete|Move|Copy|Replace|Create)\b`
  - `\[(?:System\.)?IO\.(?:StreamWriter|FileStream)\]::new\b`
  - `\[(?:System\.)?IO\.Directory\]::(?:Delete|Move|CreateDirectory)\b`
- **建议测试**: e2e 加 fixture 覆盖 `[IO.File]::WriteAllText('task.md', 'x')` 应被 deny。

#### [P1-2] PowerShell guard 缺 Tee-Object / Invoke-WebRequest 等写文件 cmdlet 与命名参数（验证编号 V13）

- **位置**: `plugin/hooks/pre-tool-use/powershell-guard.js:33-73, 75-79`
- **问题**: `powershellOutputRedirectTargets` 只解析 `>` / `>>` shell 重定向；`commandTextLooksMutating` 不含 `Tee-Object`、`Invoke-WebRequest`、`Invoke-RestMethod`、`Start-Process`、`Compress-Archive`、`Expand-Archive`、`Export-Csv`、`Export-Clixml`。`Tee-Object -FilePath task.md`、`Invoke-WebRequest https://x.com/y -OutFile task.md` 路径全部 PASS。
- **路径追踪**: 同 V12，三分支全 false。
- **影响**: 同 V12，覆盖完整性问题。
- **建议修复**:
  - 把 mutating regex 扩展为 `Set-Content|Add-Content|Out-File|Tee-Object|New-Item|Remove-Item|Move-Item|Copy-Item|Clear-Content|Rename-Item|Set-Item|Remove-ItemProperty|Set-ItemProperty|Invoke-WebRequest|Invoke-RestMethod|Export-Csv|Export-Clixml|Compress-Archive|Expand-Archive|Start-Process|Start-Job|mkdir|rmdir|rm|del|erase|mv|move|cp|copy|ni|sc|ac|tee|iwr|irm|curl|wget|saps`
  - 新增 `powershellNamedParameterTargets(command)` 解析 `-FilePath / -OutFile / -Path / -LiteralPath / -Destination / -Append` 后的目标文件，加入 `powershellCommandRedirectsToArtifact` 与 `powershellCommandRedirectsToArtifactRuntimeControl` 判定
- **建议测试**: e2e 加 `Tee-Object -FilePath task.md` 与 `Invoke-WebRequest -OutFile task.md` 应被 deny。

#### [P1-3] Bash heredoc body 在 mutating-text / reference 检查路径上未 strip（验证编号 V4）

- **位置**: `plugin/hooks/pre-tool-use/bash-guard.js:102-104, 248-250, 300-312`
- **问题**: 只有 `bashOutputRedirectTargets` (line 108) 调用 `stripHeredocBodies`；`bashCommandLooksMutating`、`bashCommandReferencesArtifact`、`bashShellCommandReferencesArtifactRuntimeControl`、`commandTextContainsWriteApi` 都对原始命令做正则匹配。
- **路径追踪**: 构造 `cat > readme.md <<'EOF'\n执行 rm task.md，详见 .pace/locks/...\nEOF`：
  - `bashCommandRedirectsToArtifact`: redirect 走 strip 路径 → 检查 `readme.md` → 不是 artifact → false
  - `bashCommandLooksMutating`: 整段命令含 `rm`（body 内） → true
  - `bashCommandReferencesArtifact`: 整段命令含 `task.md` / `.pace/locks/`（body 内） → true
  - 合成 `(LooksMutating && References)` → true → 误报 deny
- **影响**: AI 写 README、教程、commit message body 等文档时引用 PaceFlow 路径示例可能被误拦。中频但实际发生概率低。
- **建议修复**: 在 `bashCommandLooksMutating` / `bashCommandReferencesArtifact` / `bashShellCommandReferencesArtifactRuntimeControl` / `commandTextContainsWriteApi` 入口统一先 `stripHeredocBodies`，与 `bashOutputRedirectTargets` 保持对称。同步处理 PowerShell `stripHereStrings` 在对应函数入口的覆盖。
- **建议测试**: e2e 加 `cat > readme.md <<'EOF' ... rm task.md ... EOF` 应 PASS（不应被误判为 mutating artifact）。
